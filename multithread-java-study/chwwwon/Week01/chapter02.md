# 2장. 개략적인 규모 추정

- 개략적인 규모 추정: 보편적으로 통용되는 성능 수치상에서 사고 실험을 행하여 추정치를 계산하는 행위
    
    → 설계가 요구사항에 부합하는지 확인 위함
    
- 2의 제곱수, 응답지연 값, 가용성 수치 이해 필요

---

## 2의 제곱수

- 데이터 볼륨의 단위는 2의 제곱수로 표현
- 1byte = 8bit

---

### 가용성에 관계된 수치들

- 고가용성: 시스템이 오랜 시간 동안 지속적으로 중단 없이 운영될 수 있는 능력
    - 퍼센트로 표현
        
        Ex) 100%: 한 번도 중단된 적 없음
        
    - 대부분의 서비스는 99-100%를 가짐
- SLA: 서비스 사업자 - 고객 사이의 합의. 서비스 사업자가 제공하는 서비스의 가용시간 기술
    
    Ex) 대부분의 사업자(구글, 아마존, MS 등)는 99% 이상의 SLA 제공
    
    - 숫자 9를 이용하여 표현 (99%, 99.9%, 99.9999%…)

## 예제: 트위터 QPS와 저장소 요구량 추정

### 가정

- 월간 능동 사용자: 300M
- 50% 사용자가 매일 사용
- 평균적으로 매일 2건의 트윗 작성
- 미디어 포함 트윗은 10%
- 데이터는 5년간 보관

### 추정

- QPS(Query Per Second) 추정치
    - Daily Active User = 300M * 50% = 1.5M
    - QPS = 1.5M * 2 tweets / 24h / 3600s = 약 3500
    - Peek QPS = 2 * QPS = 약 700
- 미디어 저장을 위한 저장소 요구량
    - 평균 트윗 크기
        - tweet_id: 64byte
        - text: 140byte
        - media: 1MB
    - 미디어 저장소 요구량 = 1.5M * 2 * 10% * 1MB = 30TB/day
    - 5년간 미디어 보관을 위한 저장소 요구량 = 30TB * 365 * 5 = 약 55PB
