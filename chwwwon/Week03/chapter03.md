# 3장. 시스템 설계 면접 공략법

## 시스템 설계 면접

- 설계 능력의 기술적 측면 평가
- 협력 적합성
- 압박 상황에서 대처 능력
- 건설적 해결 능력

### 1. 문제 이해 및 설계 범위 확정

**요구사항과 가정들을 분명히 하라**

질문 예제

- 구체적으로 어떤 기능을 만드는가?
- 제품 사용자 수는 얼마나 되는가?
- 회사 규모? 성장 속도?
- 기술 스택?

### **2. 개략적인 설계안 제시 및 동의 구하기**

**설계안에 대한 최초 청사진을 제시하고 면접관의 의견을 구하라**

- 면접관을 팀원처럼 대하며 질문하기
- 다이어그램 작성
- 설계안과 제약사항들을 개략적으로 계산 (소리내어 설명)
- 구체적 사용 사례 살펴보기

예제

“뉴스 피드 시스템 설계하기”

- 피드 발행: 사용자 포스트 발행 → 데이터 캐시/DB 기록 → 친구의 뉴스 피드에 발행
- 피드 생성: 친구의 포스트가 시간 역순으로 정렬

### 3. 상세 설계

**설계 대상 컴포넌트 사이의 우선순위 정하기**

- 대부분의 면접관은 특정 시스템 컴포넌트들의 세부사항을 깊이 있게 설명하는 것을 보고싶어 함
- 시간 관리 주의
- 면접관에게 긍정적 신호를 전달해야 함

### 4. 마무리

**후속질문, 추가 논의 진행**

예제

- 병목구간, 개선 지점 질문 → 비판적 사고 능력 요구.
- 설계 내용 요약 → 해결책 제시한 경우 중요
- 오류 발생 따져보기
- 운영 이슈 논의
- 미래의 규모 확장 요구에 대한 대처
- 필요하지만 다루지 못한 세부 개선사항

---

+)

- 질문을 통해 확인
- 문제 요구 사항 이해
- 최선의 답은 없다
- 면접관과 소통
- 가능한 여러 해법 제시
- 면접관이 개략적 설계에 동의하면 → 각 컴포넌트의 세부사항 설명
- 면접관의 아이디어 끌어내기

-)

- 전형적인 면접 질문에 대비 후 면접장에 가라
- 요구사항이나 가정들을 분명히 하지 않은 상태에서 설계하지 말기
- 개략적 설계 후 세부사항으로 → 특정 컴포넌트에 너무 집중 X
- 진행중 막히면 힌트 요청

**시간배분**

약 45분의 시간이 주어질때

1. 문제 이해 및 설계 범위 확정 - 10분
2. 개략적 설계안 제시 및 동의 구하기: 10-15분
3. 상세 설계: 10-25분
4. 마무리: 3-5분
