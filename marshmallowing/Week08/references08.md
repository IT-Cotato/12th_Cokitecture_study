> PageRank는 Google의 창립자들이 개발한 알고리즘으로, 웹 페이지의 중요도와 권위(Authority)를 측정하는 기준이다
>

## 배경

당시 검색엔진은 단순히 **키워드**가 많이 등장하는 페이지를 상위에 노출하는 방식이었다. 인터넷에 존재하는 수많은 웹페이지 중 어떤 것이 더 유용한지를 판단하는 방법을 고민하던 중 링크 구조가 사람들의 신뢰와 추천을 드러내는 지표일 수 있다는 관점을 제시하였다.

이 발상은 웹페이지의 중요도를 링크 관계로 평가하는 **PageRank 알고리즘**으로 발전하였으며, 이후 구글 검색의 핵심 기반이 되었다.

## PageRank의 핵심 원리

PageRank는 **웹페이지의 중요도**를 계산하는 알고리즘이다. 웹의 링크 구조를 이용해, **다른 페이지들로부터 얼마나 인용(link)** 되었는가를 통해 페이지의 가치를 평가한다.

- **가중치 투표**
    - 페이지 A가 B로 링크를 걸 때, A는 B에게 자신의 PageRank 점수 일부를 양도한다.
        - 여기서 중요한 점은 **A가 가진 점수**가 A가 링크한 **모든 페이지에게 균등하게 나뉘어** 분배된다
        - 즉, **링크 수가 적은 중요한 페이지로부터 받은 링크**가 링크 수가 많은 중요하지 않은 페이지로부터 받은 링크보다 **훨씬 큰 가치**를 가짐
- **반복적 측정**
    - 모든 페이지의 초기 중요도를 동일하게 설정한 후, 이 투표 및 점수 분배 과정을 **반복**
    - 이 과정을 통해 페이지들의 중요도 점수가 더 이상 변하지 않는 **안정된 상태**로 수렴

- A, B, C 등은 페이지를 나타내고, 숫자는 PageRank를 의미
- C의 경우 B에서 링크를 걸었다는 것만으로도 PageRank값이 높게 책정됨을 볼 수 있다

### 적용 예시

- **유명 블로그나 언론사**가 내 블로그를 링크한다면 → 내 PageRank가 높아진다
- 그러나 그 블로그가 **수많은 링크를 동시에** 걸었다면 → 내 블로그에 주는 영향력은 작아진다
- **Stanford.edu** 같은 기관 사이트가 링크를 걸면 PageRank 상승 폭이 매우 크다

  → 예전에는 이를 돈 받고 거래하는 사례도 있었지만, 현재 구글이 이를 철저히 제재한다.


이처럼 **링크의 질이 수량보다 중요하다**는 것이 핵심이다

### 기본 수식

***PR(A) = (1-d) + d (PR(T1)/C(T1) + … + PR(Tn)/C(Tn))***

- **PR(A)** : A 페이지의 PageRank 값
- **T1, T2 … Tn** : A를 링크하는 다른 페이지들
- **PR(T1)** : T1 페이지의 PageRank
- **C(T1)** : T1이 가진 전체 링크 개수 (T1이 나누어주는 영향력의 수)
- **d (Damping Factor)** : 사용자가 링크를 따라갈 확률 (보통 0.85로 설정됨)

어떤 페이지 A의 페이지 랭크는 그 페이지를 인용하고 있는 다른 페이지 T1, T2, T3, .. 가 가진 페이지 랭크를 정규화시킨 값의 합

- 페이지 A의 페이지 랭크는 A라는 페이지를 가리키고 있는 다른 페이지의 페이지 랭크값이 높을수록 (더 중요할수록) 더 높아진다
- 랭크의 단순 합산은 아니기 때문에 정규화시킨 값의 합이라고 표현
- T1의 페이지 랭크가 높다고 하더라도 그 페이지에서 링크를 수천 개 달아놓았다면(C(T1)값이 높다면) 그 페이지가 기여하는 비중은 낮아진다

이러한 방식으로 페이지마다 랭크가 정해지고 이는 recursice(재귀적)으로 작동하는 재귀 호출 알고리즘이다

- 각 서버 노드는 자신이 담당하는 **부분 그래프**의 연산을 수행하며, 계산된 PageRank 값은 네트워크를 통해 다른 노드와 **합산(reduce)**된다. 이후 합산된 결과는 다시 각 노드로 **재분배(map)** 되어 다음 반복에 사용된다
- PageRank는 웹 전체를 하나의 거대한 그래프로 보고, 이를 분산 환경에서 동시에 처리하는 **대규모 확률행렬 기반 그래프 연산 시스템**이라 할 수 있다

## 랜덤 서퍼 모델

### PageRank 계산의 안정성 보장

> PageRank가 웹 전체에 대해 안정적인 점수를 계산할 수 있는 핵심 이유는 **랜덤 서퍼(Random Surfer) 모델**을 도입하여 수학적 불안정성을 해소했기 때문이다
>

- **랭크 싱크 문제 해결**
    - 순수한 링크 구조만으로는 랭크 싱크(Rank Sink) 문제가 발생
    - **랭크 싱크(Dead End):** 외부로 나가는 링크가 없는 페이지는 자신이 받은 중요도 점수를 **웹 전체로 흘려보내지 않고** 흡수해 버린다. 이는 전체 웹 그래프에서 중요도 점수가 **사라지는(Sink)** 현상을 만듦

- **댐핑 팩터 (Damping Factor)**

  > 사용자가 85% 확률로 다른 페이지로 이동하고, 15% 확률로 탐색을 멈춘다는 가정
  >
    - 이 모델은 사용자가 링크를 따라 이동하는 행위 외에, **무작위로 다른 페이지로 점프할 확률**
        - d = 1 → 사용자가 무한히 링크를 따라간다 (비현실적)
        - d = 0 → 한 페이지만 보고 멈춘다 (의미 없는 결과)
    - **무작위 점프:** 사용자는 약 **15%의 확률**로 현재 페이지의 링크를 무시하고 웹의 아무 페이지로 이동
    - **점수의 재분배:** 이 무작위 점프 덕분에 갇혔던 점수(랭크 싱크)가 웹 전체의 모든 페이지에 **균등하게 다시 분배된다**
        - 이로 인해 어떤 페이지도 점수를 완전히 흡수하거나 0으로 수렴하지 않게 되며, PageRank 점수가 항상 **안정적인 값으로 수렴**하도록 보장

1. 대부분의 사용자는 링크를 따라 이동한다.
2. 일정 확률로는 아무 페이지로 점프한다.

> 이 두 원리를 조합하여 웹 전체의 중요도를 계산한다. PageRank는 결국 **링크를 통한 신뢰의 흐름과 우연한 탐색의 균형**을 수학적으로 표현한 모델
>

## 의의와 영향력

- **빅데이터 처리의 촉매제**
    - 웹 전체의 PageRank를 계산하는 것은 수십억 페이지에 대한 **거대한 행렬 연산**을 수십 번 반복해야 하는 작업이다.
    - 이러한 대규모 병렬 계산 문제를 해결하기 위해 MapReduce와 같은 **분산 컴퓨팅 프레임워크**가 탄생하는 직접적인 계기
        - 각 페이지는 자신이 가진 랭크를 outlink 들에게 나누어 주는 과정 → map
        - 모든 페이지가 받은 랭크를 합산하는 과정 → reduce
- **그래프 분석의 기반**
    - PageRank가 확립한 ‘**연결 구조를 활용하여 노드의 영향력을 측정한다**’는 원칙은 오늘날 소셜 네트워크 분석, 추천 시스템, 그리고 지식 그래프 구축 등 복잡한 그래프 데이터 분석의 기본 아이디어
- **검색 품질의 혁명**
    - PageRank는 스팸 페이지를 걸러내고 신뢰할 수 있는 권위 있는 결과를 상단에 노출시킴으로써, 구글이 검색 시장을 지배하는 결정적인 요인이 되었다
    - 이는 **키워드 빈도 기반의 구식 검색**에서 **신뢰도 및 권위 기반의 현대적 검색**으로 패러다임을 전환시킨 사건

PageRank는 단독으로 존재하는 알고리즘이 아니라 **크롤러(Crawler) – 인덱서(Indexer) – 랭커(Ranker)** 로 이어지는 검색엔진 파이프라인의 한 구성 요소이다

- **웹 크롤러(Web Crawler)** 가 전 세계 웹페이지를 수집한다
- 각 페이지의 **링크 구조(Link Graph)** 를 추출하여 거대한 그래프 데이터로 저장한다
- 이 그래프를 기반으로 PageRank 계산이 수행된다

즉, PageRank는 링크 데이터베이스 위에서 동작하는 **그래프 처리 엔진**의 일부이다.
